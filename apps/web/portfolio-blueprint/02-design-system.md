# Bilal ESSATTE — Design System

> Version 1.0

---

# Philosophy

The design system exists to create consistency.

Every page should feel like it belongs to the same product.

The design should never compete with the work.

Projects are always the hero.

---

# Design Principles

## Smart

Every element should have a reason.

No decoration for the sake of decoration.

---

## Minimal

Reduce visual noise.

Maximize clarity.

---

## Premium

The experience should feel polished.

Not flashy.

---

## Functional

Every interaction should improve usability.

---

## Timeless

Avoid trendy effects that will look outdated in two years.

Goal:

Typography should become the main visual identity.

Hierarchy should be obvious.

Large confident headlines.

Comfortable reading.

No excessive font weights.

# Animations

Rule:

Animation must improve understanding.

Never animate because it looks cool.

---

Good animations

Hover effects

Fade

Slide

Scale

Image reveal

Smooth page transitions

Micro interactions

---

Avoid

Particles

Flying objects

Constant movement

Autoplay effects

Animations that slow navigation

---

Fast.

# Things We Never Do

Random gradients.

Glow everywhere.

Glassmorphism everywhere.

Huge drop shadows.

Animated backgrounds.

Dozens of accent colors.

Random icon styles.

Three different button styles.

Different spacing on every page.

---

# Before Adding Any New Component

Ask:

Does this improve understanding?

Does this strengthen the hierarchy?

Does this help the user?

Does it match the design system?

If not,

Don't add it.
